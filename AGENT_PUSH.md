# 🛡️ ПРАВИЛА ПУШЕЙ: AGENT_PUSH.md

Привет, коллега! Я — ИИ-Агент. Если ты тоже ИИ, либо человек, который собирается делать коммит и пушить код в репозиторий Frontend-части, **ТЫ ОБЯЗАН** прочитать и выполнить эти правила перед тем, как нажать на спусковой крючок `git push`. 

Этот документ создан кровью, слезами и сломанными билдами Vercel.

## 1. 🏷 Версионирование в коммитах (Commit Versioning)
Это **САМОЕ ГЛАВНОЕ** правило. При <u>КАЖДОМ</u> коммите (и последующем пуше) агент **ОБЯЗАН** обновлять версию проекта в `package.json` (`npm version patch` или вручную) и обязательно указывать эту новую версию + текущее время в тексте коммита!
*   **Плохо**: `fix: update header padding`
*   **Ужасно**: `fix(ui) [v3.21.2]: update again` (если v3.21.2 уже была закоммичена прошлой итерацией)
*   **Хорошо**: `fix(ui) [v3.21.3] 14:35: update header padding`
Наличие УНИКАЛЬНОЙ версии и времени в логах невероятно важно, чтобы было понятно, обновился ли билд в Vercel или кэш браузера всё ещё держит старую версию.

## 2. 🧪 Базовые проверки перед коммитом
*   Никогда не делай пуш, если ты не уверен в синтаксической корректности кода.
*   Проверь, не отрезал ли ты кусок кода, скобок `}` или `return (` при автоматической замене. 
*   **Сломанный билд — это катастрофа для продакшена.**

## 3. 🤐 Взаимодействие с Backend и API
Перед тем, как вносить изменения в структуру отправляемых (POST/GET) данных на бэкенд, **трижды подумай**:
*   Например, Телеграм виджет ожидает `hash` и `id` строго в корне объекта. Если ты попытаешься улучшить структуру и обернешь данные в `data: { user: ... }`, бэкенд рухнет с ошибкой `Missing required fields (hash, id)`.
*   Всегда соблюдай **обратную совместимость**. Если бэкенд ждет "плоский" объект — отправляй "плоский".

## 4. 🪤 Firebase и Google Auth (Осторожно!)
Аутентификация — самая хрупкая часть системы.
*   **Не ставь жесткие тайм-ауты (Promise.race) на `getRedirectResult(auth)`**. Инициализация Google Auth через редирект в мобильных браузерах (Safari/Chrome на iOS/Android) может длиться 5-10 секунд. Оборванный тайм-аутом промис ломает авторизацию Гугла полностью!

## 5. ✅ Чек-лист финального пуша
1. [ ] Убедился, что код синтаксически валиден.
2. [ ] Убедился, что версия вписана в сообщение коммита `[vX.Y.Z]`.
3. [ ] Написал осмысленное описание изменений, а не просто "fixes".
4. [ ] Выполнил `git add .`, `git commit -m "..."`, `git push`.
